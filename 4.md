## 정의

📌 논리적 모델을 실제 **DBMS에 맞게 구현 가능한 형태로 변환**하는 과정입니다.

즉, 설계된 구조를 “실제로 동작하는 데이터베이스”로 만드는 단계입니다.

---

## 🧩 주요 작업

1️⃣ DBMS별 데이터 타입 및 문법 정의

2️⃣ 인덱스, 파티션, 제약조건 등 성능 요소 고려

3️⃣ DDL 스크립트 작성 (CREATE TABLE 등)

결과물: **물리적 스키마 (Physical Schema)**

---

## 💻 예시: 테이블 생성 (MySQL 기준)

```sql
CREATE TABLE User (
    user_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL
);

CREATE TABLE Post (
    post_id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    FOREIGN KEY (user_id)
        REFERENCES User(user_id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

```

---

## 💻 PostgreSQL 버전

PostgreSQL에서는 `SERIAL` 키워드를 사용합니다.

```sql
CREATE TABLE "User" (
    user_id SERIAL PRIMARY KEY,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL
);

CREATE TABLE "Post" (
    post_id SERIAL PRIMARY KEY,
    user_id INT NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    FOREIGN KEY (user_id)
        REFERENCES "User"(user_id)
        ON DELETE CASCADE
        ON UPDATE CASCADE
);

```

---

## ⚙️ RDBMS별 차이점

| 항목 | MySQL | PostgreSQL |
| --- | --- | --- |
| 자동 증가 | AUTO_INCREMENT | SERIAL |
| 대소문자 구분 | 구분하지 않음 | 구분함 |
| 문자 인코딩 | utf8mb4 권장 | UTF8 기본 |
| 외래키 제약 | CASCADE / RESTRICT | 동일하나 더 세밀한 설정 가능 |

## 🔍 인덱스(Index) 설계

인덱스는 검색 성능 향상을 위해 **자동 또는 수동 생성**됩니다.

- 기본키(PK)는 자동 인덱스 생성
- 자주 검색되는 컬럼은 **보조 인덱스(Secondary Index)** 추가 가능

예:

```sql
CREATE INDEX idx_user_email ON User(email);

```

💡 인덱스는 검색 속도를 향상시키지만,

INSERT/UPDATE 시에는 오히려 부하가 증가할 수 있으므로 균형이 중요합니다.

---

## 🧠 INT vs BIGINT 기본키 선택

| 타입 | 크기 | 범위(UNSIGNED) | 특징 |
| --- | --- | --- | --- |
| INT | 4바이트 | 0 ~ 4,294,967,295 | 약 42억 개까지 저장 가능 |
| BIGINT | 8바이트 | 0 ~ 18,446,744,073,709,551,615 | 사실상 무한대 (범위 초과 걱정 없음) |

💡 **대규모 서비스(SNS, 로그 등)** 는 처음부터 BIGINT로 설정하는 것이 안전합니다.

## 🧩 데이터 타입 예시

| 컬럼 타입 | 설명 | 예시 |
| --- | --- | --- |
| VARCHAR(n) | 가변 길이 문자열 | VARCHAR(255) |
| TEXT | 대용량 텍스트 | 게시글 본문 |
| INT / BIGINT | 정수형 ID, 카운트 | user_id |
| DATETIME | 날짜 및 시간 | created_at |
| BOOLEAN | 논리값 | is_active |
