## 분산 시스템

**데이터가 하나의 서버에만**분산 시스템

**데이터가 하나의 서버에만 있는 것이 아니라, 여러 지리적으로 분산된 노드(서버)에 나누어 저장되고 관리되는 데이터베이스 시스템**을 말해.

쉽게 말하면

**한 곳에 모든 데이터를 몰아넣지 않고, 여러 서버가 나누어 맡아 관리하는 구조**야.

## ✅ 분산 데이터베이스 시스템의 핵심 특징

### 1) **데이터가 여러 곳에 저장**

- 서울 서버 / 부산 서버 / 해외 서버 등 다양한 위치에 데이터가 있음
- 하지만 사용자에게는 **하나의 데이터베이스처럼 보임**

### 2) **투명성(Transparency)**

사용자는 데이터가 어디에 있는지 몰라도 됨

→ 시스템이 자동으로 찾아서 조회/수정해 줌

### 3) **고가용성(Availability)**

- 한 서버가 고장 나도, 다른 서버가 서비스 계속 제공
- 장애에 강함

### 4) **성능 향상(Performance)**

- 여러 서버가 동시에 처리
- 읽기/쓰기 부하를 분산

### 5) **확장성(Scalability)**

- 서버를 추가하는 것만으로 시스템 확장 가능 (수평 확장)

## 🔧 분산 방식 (중요!)

### **① 데이터 분할(Sharding)**

- 데이터를 고객 ID 기준으로 나눠서 여러 서버에 저장하는 방식
    
    예:
    
- 서버 A: ID 1–10000
- 서버 B: ID 10001–20000

대형 서비스(카카오, 인스타그램, 유튜브)에서 많이 사용

### **② 데이터 복제(Replication)**

- 동일한 데이터를 여러 노드가 가지고 있음
- 주로 읽기 성능 향상 + 장애 대비 목적
    
    예:
    
- Master(쓰기)
- Replica(읽기)

## ⭐ 장점

- **장애 내성(Fault tolerance)**
- **성능 향상**
- **트래픽 분산**
- **확장성 우수**

## ❗ 단점

- **일관성 유지가 어려움(CAP theorem 관련)**
- 설계/운영 **복잡**
- 트랜잭션 관리가 까다로움 (분산 트랜잭션)
